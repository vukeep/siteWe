# StudioWe — План реализации

## Фаза 0. Подготовка (1 неделя)
- Утвердить структуру экранов (Structure.md) и copy.
- Выбрать CMS (Sanity vs Strapi/Payload) и хостинг.
- Зафиксировать дизайн-направление, moodboard и бренд-гайд.
- Настроить репозиторий, CI/CD костяк, husky-хуки (lint, test).

## Фаза 1. Дизайн-система и UI-кит (1–2 недели)
- Перенести токены цвета/типографики в Tailwind и `tokens.json`.
- Развернуть shadcn/ui, создать базовые компоненты (Button, Card, Typography, Accordion, Form Controls, Dialog).
- Настроить Storybook для визуального контроля и документации компонентов.
- Согласовать состояния компонентов (hover, focus, dark mode при необходимости).

## Фаза 2. CMS и данные (1 неделя)
- Смоделировать схемы: hero, benefits, services grid, portfolioItem, package, faq, contactInfo, leadForm.
- Реализовать контент-студию (если Sanity — desk structure; если Strapi — collection types).
- Настроить webhooks для revalidate `/api/revalidate`.
- Заполнить стартовый контент (5–10 портфолио, тарифы, FAQ).

## Фаза 3. Фронтенд: каркас страниц (2 недели)
- Создать маршруты Next.js (`app/page.tsx`, `app/portfolio/[slug]/page.tsx`, `app/api/lead/route.ts`).
- Реализовать data-fetching (SSG + ISR) для главной, тарифов и портфолио.
- Собрать лендинг по секциям:
  - Hero + CTA,
  - Проблема/решение,
  - Форматы роликов (плитка),
  - Галерея вертикальных видео,
  - Процесс (7 шагов, анимации),
  - Тарифы,
  - Преимущества + миссия,
  - FAQ,
  - Контактная форма.
- Добавить SEO-метаданные (OG, Schema.org, breadcrumbs).

## Фаза 4. Интерактив и анимации (1 неделя)
- Подключить Motion для появления блоков, скролл-анимированных таймлайнов, слайдера галереи.
- Добавить ленивую загрузку превью Cloudinary.
- Настроить плавные переходы между разделами (scroll-to anchors, sticky навигация).

## Фаза 5. Формы и интеграции (1 неделя)
- Реализовать React Hook Form + Zod схемы.
- API-роут: запись лидов (CRM/Notion/Sheets) + email/Telegram уведомления.
- Подключить reCAPTCHAv3, обработку ошибок и состояния (pending, success, fail).
- Настроить событийную аналитику: клики по CTA («Посмотреть примеры», «Хочу также», «Получить расчёт», форма), источник хранилища (Posthog/Snowplow) и дашборды.

## Фаза 6. Тестирование и оптимизация (1 неделя)
- Линты/типы, unit-тесты критичных компонентов, e2e для формы (Playwright).
- Lighthouse и WebPageTest → оптимизация (preload шрифтов, размер бандла).
- Проверить адаптивность: мобильные вертикальные видео, тарифы, таймлайн.
- QA-проверка контента с CMS (предпросмотр, ISR).

## Фаза 7. Деплой и сопровождение (1 неделя)
- Собрать Docker образ (`next build`, `next export` не требуется, режим standalone).
- Настроить Nginx (SSL, redirects, caching headers).
- Прогнать smoke-тесты на staging → production.
- Если выбрана self-hosted CMS (Strapi/Payload) — задеплоить отдельный сервис/кластер, настроить бэкапы БД/медиа и политику обновлений; при выборе Sanity — зафиксировать окружения и доступы.
- Обучить контент-команду работе с CMS, регламент обновления портфолио и тарифов.

## Контур поддержки
- Ретроспектива и backlog улучшений (мультиязычность, блог, генерация лид-магнитов).
- Мониторинг Sentry + uptime; SLA ответа на баги от форм ≤24ч.
